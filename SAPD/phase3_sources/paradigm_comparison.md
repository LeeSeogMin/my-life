# 패러다임 비교: 증거 기반 정책(EBP) vs. 상태 기반 적응형 정책(SAPD)

## 1. 개요

본 문서는 증거 기반 정책(Evidence-based Policy, EBP)과 상태 기반 적응형 정책(State-based Adaptive Policy, SAPD)을 10개 이상의 분석 차원에 걸쳐 체계적으로 비교한다. 이 비교는 P1(EBP의 구조적 한계) 논증의 핵심 근거 자료로 활용된다.

---

## 2. 종합 비교표

| 번호 | 비교 차원 | 증거 기반 정책 (EBP) | 상태 기반 적응형 정책 (SAPD) |
|------|----------|---------------------|--------------------------|
| 1 | **인식론적 기반** | 사후적 실증주의: 과거 데이터와 실험 결과에 기반한 증거가 정책 결정의 근거 | 실시간 상태주의: 현재 시스템 상태(State(t))가 의사결정의 직접적 입력 |
| 2 | **시간 모델** | 분절적·순차적: 정책 → 실행 → 평가 → 수정의 선형 주기 | 연속적·순환적: State(t) → Decision(t) → Response → State(t+1)의 항상적 순환 |
| 3 | **피드백 구조** | 분절적 피드백 루프: 평가가 정책 주기의 마지막 단계에 위치, 각 단계가 독립적 절차로 작동 | 연속적 피드백 루프: 상태 관찰과 평가가 동시에 진행, 학습이 상시적 과정 |
| 4 | **데이터 활용 방식** | 스냅샷(Snapshot): 특정 시점의 데이터 수집·분석, 데이터가 보고서 형태로 소비됨 | 흐름(Flow): 실시간 데이터 스트림의 연속적 관찰, 데이터가 상태 변수로 시스템에 통합됨 |
| 5 | **정책의 존재론적 지위** | 개입(Intervention): 정책은 특정 문제에 대한 고정된 처방 | 상태 함수(State Function): 정책은 현재 상태에 따라 결정을 산출하는 동태적 함수 |
| 6 | **적응 메커니즘** | 사후적 수정: 평가 결과에 기반한 정책 수정 (주기적, 지연적) | 실시간 적응: 상태 변화에 따른 의사결정 규칙의 연속적 갱신 |
| 7 | **AI의 역할** | 분석 도구: 증거 생산을 위한 데이터 분석 보조 도구 | 시스템 구성 요소: 의사결정 시스템의 적응력을 강화하는 아키텍처 내 구조적 요소 |
| 8 | **설계 수준** | 원리 수준: "증거에 기반하라"는 규범적 원칙 | 아키텍처 수준: 5단계 레이어 구조의 구체적 설계 명세 |
| 9 | **학습 유형** | 단일 루프 학습: 평가 결과에 따른 정책 내용의 수정 | 이중 루프 학습: 정책 내용뿐 아니라 의사결정 규칙(f) 자체의 진화 |
| 10 | **시간 지연 대응** | 단축 전략: 평가 속도 향상, 데이터 분석 기술 발전을 통한 지연 '단축' | 구조 전환 전략: 피드백 루프 자체를 연속적으로 재설계하여 시간 지연 문제 자체를 '해소' |
| 11 | **평가의 위치** | 종단 위치: 정책 주기의 마지막 단계 | 편재 위치: 상태 관찰과 동시에 수행되는 상시적 과정 |
| 12 | **불확실성 대응** | 증거의 축적: 더 많은, 더 좋은 증거로 불확실성을 줄임 | 적응력 확보: 불확실성을 제거하려 하지 않고, 불확실성 속에서 적응하는 시스템을 설계함 |
| 13 | **환경 인식** | 상대적 안정성 전제: 증거가 유효한 기간 동안 환경이 크게 변하지 않는다는 암묵적 가정 | 동태성 전제: 환경은 지속적으로 변화하며, 시스템은 이에 실시간으로 대응해야 함 |
| 14 | **핵심 질문** | "과거에 무엇이 작동했는가?" (What worked?) | "현재 상태에서 무엇이 작동해야 하는가?" (What should work now?) |
| 15 | **실패 모드** | 시간 부적합(Temporal Mismatch): 과거의 증거가 현재의 상태에 부적합 | 상태 오관찰(State Misperception): 상태 변수의 부정확한 관찰 또는 누락 |

## 3. 차원별 심층 분석

### 3.1 인식론적 기반 (차원 1)

EBP의 인식론적 기반은 **사후적 실증주의(retrospective positivism)**이다. 무작위통제실험(RCT), 메타분석, 체계적 문헌 검토 등을 통해 생산된 과거의 증거가 미래의 정책 결정을 안내한다는 논리이다. 이 접근의 강점은 방법론적 엄밀성이며, 약점은 과거의 증거가 현재/미래의 맥락에서 유효하다는 가정이 충족되지 않을 때 발생한다(Cartwright & Hardie, 2012).

SAPD의 인식론적 기반은 **실시간 상태주의(real-time statism)**이다. 의사결정의 근거는 과거의 실험 결과가 아니라 현재 시점의 시스템 상태(State(t))이다. 과거 증거가 무용하다는 것이 아니라, 과거 증거는 의사결정 규칙 f의 설계에 반영되되, 의사결정 자체는 현재 상태를 입력으로 실시간으로 산출된다.

### 3.2 시간 모델 (차원 2)

EBP의 시간 모델은 **분절적·순차적(fragmented and sequential)**이다. 정책 설계, 실행, 평가, 수정이 각각 독립된 시간 블록으로 작동하며, 한 블록이 완료되어야 다음 블록이 시작된다. 이 구조에서 전체 주기의 소요 시간이 곧 시간 지연(τ)이 된다.

SAPD의 시간 모델은 **연속적·순환적(continuous and cyclical)**이다. State(t) → Decision(t) → Response → State(t+1)의 순환이 끊임없이 반복되며, 평가는 별도의 시간 블록이 아니라 상태 관찰의 일부로 통합된다. 이론적으로 시간 지연(τ)은 상태 관찰 속도와 의사결정 규칙의 연산 속도에 의해 결정되며, 피드백 구조 자체의 분절에 의한 지연은 해소된다.

### 3.3 정책의 존재론적 지위 (차원 5)

EBP에서 정책은 **개입(intervention)**이다. 특정 문제에 대한 특정 처방으로서, 설계된 이후에는 (평가에 의해 수정될 때까지) 고정적으로 실행된다. 정책의 질은 설계 시점의 증거의 질에 의존한다.

SAPD에서 정책은 **상태 함수(state function)**이다. `Decision(t) = f(State(t))`로 표현되는 이 함수는 현재 상태에 따라 다른 의사결정을 산출하는 동태적 시스템이다. 정책의 질은 의사결정 규칙 f의 설계 품질과 상태 관찰의 정확성에 의존한다.

이 존재론적 전환은 근본적이다. 정책을 '개입'에서 '함수'로 재정의하면, 정책 설계의 초점이 "어떤 개입을 할 것인가"에서 "어떤 의사결정 시스템을 구축할 것인가"로 전환된다. 이것이 SAPD가 '의사결정 아키텍처 설계론'을 지향하는 근본적 이유이다.

### 3.4 불확실성 대응 (차원 12)

EBP의 불확실성 대응 전략은 **증거의 축적**이다. 더 많은 연구, 더 엄밀한 방법론, 더 포괄적인 체계적 검토를 통해 불확실성을 점진적으로 줄여나간다. 이 접근은 불확실성이 '지식의 부족'에서 기인한다는 전제에 기반한다.

SAPD의 불확실성 대응 전략은 **적응력의 확보**이다. 불확실성은 완전히 제거될 수 없는 환경의 본질적 속성이며, 따라서 불확실성을 줄이려는 것이 아니라 불확실성 속에서 적응하는 시스템을 설계하는 것이 목표이다. 이는 적응적 거버넌스(Folke et al., 2005)의 회복력(resilience) 관점과 궤를 같이하며, SAPD는 이를 설계 가능한 아키텍처로 구체화한다.

### 3.5 실패 모드 (차원 15)

각 패러다임의 '실패 모드'를 분석하는 것은 해당 패러다임의 구조적 약점을 이해하는 데 유용하다.

**EBP의 실패 모드 -- 시간 부적합(Temporal Mismatch)**: 증거가 생산된 시점과 정책이 적용되는 시점 사이에 환경이 변화하여, 과거의 증거가 현재의 문제에 부적합해지는 경우이다. 이는 EBP의 구조적 시간 지연에서 기인하며, 환경 변화 속도가 빠를수록 발생 확률이 높아진다.

**SAPD의 실패 모드 -- 상태 오관찰(State Misperception)**: 상태 변수가 부정확하게 관찰되거나 핵심 상태 변수가 누락되어, 의사결정 규칙이 잘못된 상태 정보에 기반하여 작동하는 경우이다. 이는 Layer 1(문제 구조화)에서의 상태 변수 선정과 Layer 4(데이터 통합)에서의 관찰 인프라의 품질에 의존한다.

## 4. 비교의 함의

### 4.1 EBP의 가치 인정
본 비교는 EBP의 가치를 부정하지 않는다. EBP는 정책 결정에서 이념과 관행을 넘어 체계적 증거를 활용하는 규범적 전환을 이끌었으며, 이 기여는 영구적이다. SAPD는 EBP를 '대체'하는 것이 아니라 EBP의 인식론적 성과를 계승하면서 구조적 한계를 극복하는 '발전적 전환'이다.

### 4.2 패러다임 전환의 필요성
비교에서 도출되는 핵심 함의는, EBP에서 SAPD로의 전환이 단순한 기술적 개선이 아니라 **인식론적·존재론적 전환**이라는 것이다. 정책의 존재론적 지위가 '개입'에서 '함수'로, 증거의 역할이 '근거'에서 '상태 입력'으로, 평가의 위치가 '종단'에서 '편재'로 전환되며, 이는 정책 설계의 근본적 패러다임 전환을 의미한다.

### 4.3 공존 가능성
두 패러다임이 완전히 배타적인 것은 아니다. 안정적 환경에서의 장기 정책(예: 교육 과정 개편)에는 EBP의 체계적 증거 축적이 여전히 유효하다. 동태적 환경에서의 적응적 정책(예: 감염병 대응, 재난 관리, 실시간 교통 관리)에는 SAPD의 연속적 피드백 구조가 더 적합하다. SAPD는 EBP가 적용될 수 없는 영역을 커버하는 보완적 패러다임으로 기능할 수 있으며, 동시에 EBP의 성과를 자체 아키텍처 내에 흡수할 수 있다(의사결정 규칙 f의 설계에 과거 증거를 반영).

## 5. 비교 요약 다이어그램

```
EBP 모델:
  [정책 설계] → [실행] → [데이터 수집] → [분석] → [평가 보고서] → [정책 수정]
       ↑_______________________________________________|
                    (τ: 수개월 ~ 수년의 시간 지연)

SAPD 모델:
  [State(t)] → [f(State(t)) = Decision(t)] → [System Response] → [State(t+1)]
       ↑___________________________________________________|
                    (τ → 최소화: 연속적 순환)
```
