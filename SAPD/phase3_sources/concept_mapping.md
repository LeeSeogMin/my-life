# 개념 매핑: SAPD 레이어와 기존 이론적 개념의 대응 관계

## 1. 개요

본 문서는 SAPD Framework의 5단계 아키텍처 각 레이어가 기존의 이론적 개념들과 어떤 관계에 있는지를 체계적으로 매핑한다. 이 매핑은 (1) SAPD가 기존 이론을 무시한 독립적 구성이 아닌 학술적 전통의 계승·발전임을 보여주고, (2) 기존 이론들이 커버하지 못한 영역을 SAPD가 어떻게 충당하는지를 명시하기 위한 것이다.

---

## 2. Layer 1: 문제 구조화 (Problem Structuring)

### SAPD에서의 기능
시스템의 경계와 정책 문제를 정의하며, 상태 변수(State Variables)의 범위를 설정한다. "무엇이 문제이며, 시스템의 경계는 어디까지인가?"에 답한다.

### 기존 이론과의 매핑

| 기존 이론/개념 | 대응 개념 | 관계 유형 | 설명 |
|--------------|----------|----------|------|
| **적응적 거버넌스** (Dietz et al., 2003) | 시스템 경계 정의, 공유자원의 범위 설정 | 계승 | 적응적 거버넌스에서의 시스템 경계 설정 원리를 정책 전 영역으로 확장 |
| **사이버네틱스** (Ashby, 1956) | 피제어 시스템의 정의, 필요다양성의 범위 설정 | 계승 | Ashby의 시스템 경계 정의 원리를 공공 정책으로 구체화 |
| **VSM** (Beer, 1972) | System 1 (운영 단위)의 정의 | 부분적 대응 | Beer의 운영 단위 정의가 문제 구조화의 출발점에 해당 |
| **DAPP** (Haasnoot et al., 2013) | 문제 영역(problem domain)의 정의 | 부분적 대응 | DAPP의 문제 영역 설정은 특정 도메인(기후 등)에 한정되나, L1은 범용적 |
| **정책 분석** (Dunn, 2018) | 문제 구조화(problem structuring) | 직접 대응 | 정책 분석의 문제 구조화 개념을 시스템 이론적 관점에서 재해석 |
| **팔란티어** | 온톨로지(Ontology): 개체·관계 정의를 통한 문제 도메인 설정 | 구조적 동형 | 이론적 '문제 구조화'가 기술적 '온톨로지 설계'로 구현됨 |

### SAPD의 추가적 기여
기존의 문제 구조화 개념들은 일회적(one-time) 과정으로 다루어지는 경향이 있다. SAPD에서 Layer 1은 Layer 5(적응형 피드백)와 직접 연결되어, 문제 정의 자체가 피드백에 의해 재정의될 수 있는 순환적 과정이다.

---

## 3. Layer 2: 전략 구조화 (Strategy Structuring)

### SAPD에서의 기능
전략적 구조와 성과 생성 논리(theory of change)를 정의하며, 목표 상태(desired state)를 설정한다. "어떤 방향으로, 어떤 논리로 변화를 이끌 것인가?"에 답한다.

### 기존 이론과의 매핑

| 기존 이론/개념 | 대응 개념 | 관계 유형 | 설명 |
|--------------|----------|----------|------|
| **변화 이론** (Weiss, 1995) | 프로그램의 인과적 논리 모델 | 계승 | 변화 이론의 인과 논리를 상태 전이 함수의 목표 설정으로 재해석 |
| **논리 모형** (W.K. Kellogg Foundation, 2004) | 투입-활동-산출-성과의 논리적 연결 | 부분적 대응 | 논리 모형의 선형적 구조를 순환적 상태 전이 구조로 확장 |
| **RDM** (Lempert et al., 2003) | 전략의 강건성 평가 | 보완적 활용 | RDM의 전략 평가 방법론이 L2 내에서 전략의 강건성을 검증하는 도구로 활용 가능 |
| **DAPP** (Walker et al., 2013) | 적응 경로 설계, 조건부 행동 계획 | 부분적 대응 | DAPP의 경로 설계가 L2의 전략 구조화 내 하나의 방법론으로 포섭 가능 |
| **전략 기획** (Mintzberg, 1994) | 의도된 전략 vs. 창발적 전략 | 이론적 배경 | Mintzberg의 창발적 전략 개념이 SAPD의 적응적 전략 갱신 원리와 조응 |
| **팔란티어** | 비즈니스 규칙·프로세스: 온톨로지에 연결된 비즈니스 로직 | 구조적 동형 | 이론적 '전략 구조화'가 기술적 '비즈니스 로직 설계'로 구현됨 |

### SAPD의 추가적 기여
기존의 전략 구조화 도구들(변화 이론, 논리 모형)은 선형적이고 정적인 구조를 전제한다. SAPD의 Layer 2는 전략의 성과 논리를 상태 전이 함수의 목표 설정과 연결함으로써, 전략이 상태 변화에 따라 동태적으로 재구성될 수 있는 가능성을 구조적으로 확보한다.

---

## 4. Layer 3: 의사결정 아키텍처 (Decision Architecture)

### SAPD에서의 기능
의사결정 노드, 권한, 기준을 정의하며, 의사결정 규칙 f의 구조를 설계한다. "누가, 어떤 기준으로, 어떤 절차에 따라 결정하는가?"에 답한다.

### 기존 이론과의 매핑

| 기존 이론/개념 | 대응 개념 | 관계 유형 | 설명 |
|--------------|----------|----------|------|
| **사이버네틱스** (Ashby, 1956) | 제어 함수(control function) | 직접 계승 | Ashby의 제어 함수 개념이 의사결정 규칙 f의 이론적 원형 |
| **VSM** (Beer, 1972) | System 3 (내부 규제), System 4 (외부 지향) | 부분적 대응 | Beer의 시스템 3, 4의 조절 기능이 의사결정 아키텍처의 구성 요소에 해당 |
| **관료제 이론** (Dunsire, 1978) | 행정적 재량과 통제 메커니즘 | 이론적 배경 | 관료제의 의사결정 구조가 L3의 제도적 맥락을 형성 |
| **적응적 관리** (Walters, 1986) | 능동적/수동적 적응 의사결정 모델 | 부분적 대응 | 적응적 관리의 의사결정 모델이 L3 내 의사결정 규칙의 한 유형으로 포섭 |
| **행정 재량론** (Engstrom & Ho, 2020) | 알고리즘적 의사결정과 행정 재량의 접점 | 보완적 활용 | AI 기반 의사결정에서의 행정 재량 보장 문제가 L3의 설계 제약 조건으로 반영 |
| **팔란티어** | AIP Logic, AI Agent Studio: 의사결정 워크플로우 자동화 | 구조적 동형 | 이론적 '의사결정 아키텍처'가 기술적 '워크플로우 엔진'으로 구현됨 |

### SAPD의 추가적 기여
기존의 의사결정 이론들은 개별 의사결정의 질을 높이는 데 집중하는 반면, SAPD의 Layer 3는 의사결정이 이루어지는 구조(architecture) 자체를 설계 대상으로 삼는다. 의사결정의 노드(누가), 기준(어떤 근거로), 권한(어떤 범위에서), 절차(어떤 순서로)를 아키텍처 수준에서 명시적으로 정의하는 것이 SAPD의 고유한 접근이다.

---

## 5. Layer 4: 데이터 통합 (Data Integration)

### SAPD에서의 기능
실시간 데이터와 AI를 시스템에 통합하며, 상태 변수의 관찰 인프라를 구축한다. "어떤 데이터가, 어떻게 의사결정에 연결되는가?"에 답한다.

### 기존 이론과의 매핑

| 기존 이론/개념 | 대응 개념 | 관계 유형 | 설명 |
|--------------|----------|----------|------|
| **디지털 정부** (O'Reilly, 2011) | 정부 데이터의 개방과 활용 | 부분적 대응 | 디지털 정부의 데이터 활용 비전이 L4의 배경이나, L4는 의사결정 시스템과의 구조적 통합에 집중 |
| **데이터 생태계** (Janssen & Estevez, 2013) | 정부 데이터 생태계 구축 | 부분적 대응 | 데이터 생태계의 데이터 연결·활용 개념이 L4의 기반 인프라에 해당 |
| **공공부문 AI** (Wirtz et al., 2019) | AI 도입의 기회·위험 관리 | 보완적 활용 | 공공부문 AI의 거버넌스 논의가 L4 내 AI 배치의 제약 조건으로 반영 |
| **알고리즘 거버넌스** (Engstrom & Ho, 2020) | AI 의사결정의 적법성·투명성 | 보완적 활용 | 알고리즘 거버넌스 원칙이 L4의 AI 설계 제약 조건으로 통합 |
| **EBP** (Pawson, 2006) | 증거의 체계적 수집·활용 | 재해석 | EBP의 증거 활용 원칙을 실시간 데이터 통합으로 재해석 |
| **팔란티어** | Foundry: 이질적 소스의 실시간 데이터 중앙 통합 | 구조적 동형 | 이론적 '데이터 통합'이 기술적 '데이터 플랫폼'으로 구현됨 |

### SAPD의 추가적 기여
기존 문헌에서 데이터와 AI는 정책 시스템의 '외부에서 도입되는 자원'으로 다루어진다. SAPD의 Layer 4는 데이터와 AI를 의사결정 아키텍처의 '내부 구성 요소'로 배치하며, 상태 변수의 관찰 → 의사결정 규칙의 연산 → 시스템 반응의 측정이라는 전체 순환에서 데이터와 AI가 어떤 구조적 위치를 점하는지를 명시적으로 정의한다. AI의 역할은 "인간의 의사결정을 대체하는 것이 아니라 의사결정 시스템의 적응력을 강화하는 것"으로 한정된다.

## 6. Layer 5: 평가 및 적응형 피드백 (Evaluation & Adaptive Feedback)

### SAPD에서의 기능
지속적 평가와 적응 메커니즘을 제공하며, 시스템의 학습 순환을 완성한다. "시스템이 결과로부터 무엇을 학습하며, 어떻게 진화하는가?"에 답한다.

### 기존 이론과의 매핑

| 기존 이론/개념 | 대응 개념 | 관계 유형 | 설명 |
|--------------|----------|----------|------|
| **사이버네틱스** (Ashby, 1956) | 피드백 루프(feedback loop) | 직접 계승 | Ashby의 피드백 제어 원리가 L5의 이론적 원형 |
| **조직 학습** (Argyris & Schon, 1978) | 단일 루프/이중 루프 학습 | 계승 | L5는 의사결정 출력의 조정(단일 루프)과 의사결정 규칙 f의 갱신(이중 루프) 모두를 포괄 |
| **적응적 관리** (Holling, 1978) | 모니터링-평가-조정 순환 | 계승·확장 | 적응적 관리의 순환 학습을 실시간 연속 과정으로 확장 |
| **현실주의 평가** (Pawson & Tilley, 1997) | 맥락-메커니즘-결과 분석 | 보완적 활용 | 현실주의 평가의 분석 논리가 L5의 평가 방법론 중 하나로 활용 가능 |
| **EBP 평가 전통** | 정책 효과성 평가 | 재해석 | EBP의 사후 평가를 실시간 적응형 피드백으로 재해석 |
| **팔란티어** | 온톨로지 기록 + 학습 루프: 실행 결과의 재기록을 통한 지속적 학습 | 구조적 동형 | 이론적 '적응형 피드백'이 기술적 '온톨로지 학습 루프'로 구현됨 |

### SAPD의 추가적 기여
기존의 평가 접근들은 평가를 정책 주기의 '마지막 단계'로 위치 지정한다. SAPD의 Layer 5는 평가를 '상시적 과정(continuous operation)'으로 재정의하며, 평가의 결과가 Layer 1(문제 재정의)부터 Layer 4(데이터/AI 조정)까지 모든 레이어에 피드백되는 완전 순환 구조를 구축한다. 특히 의사결정 규칙 f 자체의 진화(이중 루프 학습)를 구조적으로 지원하는 것이 기존 평가 접근과의 핵심 차이이다.

## 8. 매핑으로부터의 핵심 발견

### 8.1 이론적 계승의 체계성
SAPD의 각 레이어는 기존 이론의 특정 개념을 체계적으로 계승하고 있다. 이는 SAPD가 학술적 진공 상태에서 탄생한 것이 아니라, 사이버네틱스, 적응적 거버넌스/관리, 정책 분석, 조직 학습 등 다양한 학술 전통의 축적된 성과 위에 구축되었음을 보여준다.

### 8.2 통합적 아키텍처의 고유성
기존의 각 이론/개념은 SAPD의 특정 레이어에 대응하는 부분적 기여를 제공하지만, 이들을 통합적 아키텍처로 구성한 것은 SAPD가 최초이다. 기존 이론들은 각각 문제 정의, 전략 설계, 의사결정 모델, 데이터 활용, 학습 등 개별 영역에 기여하되, 이들을 하나의 순환적 시스템으로 통합하지는 못했다.

### 8.3 팔란티어와의 이중 대응
SAPD의 각 레이어가 기존 이론과 대응됨과 동시에 팔란티어의 기술 요소와도 대응된다는 것은 주목할 만한 발견이다. 이는 "이론적 전통 → SAPD 아키텍처 → 기술적 구현"이라는 삼중 대응(triple correspondence) 구조를 형성하며, SAPD가 이론과 실무를 잇는 가교 역할을 수행할 수 있음을 시사한다.
