# 내적 정합성 검증 (Internal Consistency Check)

## 1. 검증 목적

본 문서는 SAPD Framework의 내적 정합성(internal consistency)을 체계적으로 검증한다. 검증의 대상은 다음 세 가지이다: (1) 각 레이어 간 논리적 흐름의 일관성, (2) 상태 전이 공식과 5-Layer 아키텍처 간의 정합성, (3) 레이어 간 잠재적 간극 또는 중복의 존재 여부.

---

## 2. 레이어 간 논리적 흐름 검증

### 2.1 L1 → L2: 문제 구조화에서 전략 구조화로

**검증 질문**: L1의 출력이 L2의 입력으로 논리적으로 충분한가?

**분석**:
- L1의 출력: 시스템 경계 정의, 상태 변수 목록 `S = {s₁, ..., sₙ}`, 개체-관계 모형, 외생 변수
- L2의 입력 요구: 시스템의 현재 상태 정보, 정책 목표 설정을 위한 도메인 경계

L1이 "시스템이 무엇으로 구성되어 있는가"를 정의하면, L2는 "이 시스템이 어떤 상태로 나아가야 하는가"를 정의한다. L1의 상태 변수 정의가 L2의 목표 상태 `S*` 정의의 전제 조건이므로, 논리적 연결은 타당하다.

**판정**: 정합적 (Consistent)

**잠재적 보완점**: L2의 정책 목표는 L1의 시스템 경계 내에서만 정의될 수 있는가? 시스템 경계 외부의 가치(예: 세대 간 형평성)가 정책 목표로 설정될 경우, L1의 경계 정의가 L2를 제약하는 것이 아니라 L2의 목표가 L1의 경계를 확장시킬 수 있다. 이는 L5→L1 피드백의 변형으로 해석 가능하나, L2→L1 역방향 영향에 대한 명시적 기술이 보완될 필요가 있다.

### 2.2 L2 → L3: 전략 구조화에서 의사결정 아키텍처로

**검증 질문**: L2에서 설계된 전략 구조가 L3의 의사결정 노드 설계를 충분히 제약하는가?

**분석**:
- L2의 출력: 목표 상태 `S*`, 변화이론, 성과 지표 `KPI`, 전략 대안 집합
- L3의 입력 요구: 어떤 의사결정이 필요한지, 그 의사결정이 어떤 상태 변수에 기반해야 하는지

L2의 변화이론이 "어떤 개입이 어떤 메커니즘을 통해 상태를 변화시키는가"를 정의하면, L3은 "그 개입을 누가, 언제, 어떤 조건에서 실행할 것인가"의 의사결정 구조를 설계한다. 전략(L2)이 의사결정(L3)의 내용을 결정하고, L3은 그 내용을 실행 가능한 노드와 규칙으로 구체화한다.

**판정**: 정합적 (Consistent)

### 2.3 L3 → L4: 의사결정 아키텍처에서 데이터 통합으로

**검증 질문**: L3의 의사결정 함수가 L4의 데이터 통합 설계를 적절히 규정하는가?

**분석**:
- L3의 출력: 의사결정 함수 `Decision(t) = f(State(t))`, 각 노드의 정보 요구
- L4의 입력 요구: 어떤 상태 변수를 관측해야 하는지, 어떤 정밀도와 빈도로 관측해야 하는지

L3이 "이 의사결정은 이 상태 변수에 기반한다"를 정의하면, L4는 "그 상태 변수를 어떻게 관측하고 시스템에 통합할 것인가"를 설계한다. 의사결정 함수의 입력 요구가 데이터 통합의 설계 명세를 결정하므로, 논리적 흐름은 타당하다.

**판정**: 정합적 (Consistent)

**잠재적 보완점**: 현실에서는 L4(데이터 가용성)가 L3(의사결정 설계)를 제약하는 역방향 영향이 강하다. 데이터가 존재하지 않는 상태 변수에 기반한 의사결정 함수는 구현 불가능하다. 이 역방향 제약은 현재 아키텍처에서 L4↔L1, L4↔L3의 교차 연결로 암시되어 있으나, 보다 명시적인 "데이터 가용성 제약(data availability constraint)" 메커니즘이 기술될 수 있다.

### 2.4 L4 → L5: 데이터 통합에서 평가 및 피드백으로

**검증 질문**: L4의 데이터 통합이 L5의 평가 및 피드백을 충분히 지원하는가?

**분석**:
- L4의 출력: 실시간 상태 벡터 `State(t)`, AI/ML 모형 출력
- L5의 입력 요구: 현재 상태와 목표 상태의 비교를 위한 데이터

L4가 상태 변수를 실시간으로 관측하면, L5는 이를 L2에서 정의된 목표 상태와 비교하여 성과 갭을 측정한다. L4의 데이터가 L5 평가의 원료이므로, 논리적 연결은 명확하다.

**판정**: 정합적 (Consistent)

### 2.5 L5 → L1: 피드백 순환의 완결

**검증 질문**: L5에서 L1으로의 피드백이 순환을 논리적으로 완결시키는가?

**분석**:
- L5의 출력: 성과 갭 평가, 의사결정 규칙 유효성 판단, 문제 재정의 트리거
- L1의 재입력 요구: 시스템 경계와 문제 정의의 재검토 신호

L5가 "현재 시스템 정의로는 목표 달성이 구조적으로 불가능하다"는 판단을 내릴 때, 이는 L1의 문제 정의 자체를 재검토하는 트리거로 기능한다. 이는 Argyris & Schon의 이중 루프 학습(double-loop learning) 개념과 일치하며, 시스템이 자기 자신의 전제를 수정하는 능력을 아키텍처 수준에서 보장한다.

**판정**: 정합적 (Consistent)

## 3. 상태 전이 공식과 5-Layer 아키텍처의 정합성

### 3.1 공식 검토

SAPD의 핵심 공식: `Decision(t) = f(State(t)) → State(t+1)`

이 공식의 각 요소가 5-Layer 아키텍처의 어느 레이어에 대응하는지를 검증한다:

| 공식 요소 | 대응 레이어 | 정합성 |
|----------|-----------|--------|
| `State(t)` — 현재 상태의 관측 | L4 (데이터 통합)에서 관측; L1 (문제 구조화)에서 정의 | 정합적 |
| `f` — 의사결정 함수 | L3 (의사결정 아키텍처)에서 설계 | 정합적 |
| `Decision(t)` — 의사결정 산출물 | L3에서 생성; L2 (전략 구조화)의 목표에 의해 방향 결정 | 정합적 |
| `State(t+1)` — 갱신된 상태 | L5 (평가 피드백)에서 관측 및 평가; L4에서 재관측 | 정합적 |
| `→` — 상태 전이 | 전체 순환 과정 자체 | 정합적 |

### 3.2 정합성 판정

핵심 공식의 모든 요소가 5-Layer 아키텍처 내에서 명확한 대응 레이어를 가지며, 어떤 요소도 아키텍처 외부에 위치하지 않는다. 공식은 아키텍처의 축약 표현(compact representation)으로 기능하며, 아키텍처는 공식의 구조적 전개(structural elaboration)로 기능한다. 양자 간 정합성은 확인된다.

### 3.3 공식의 한계에 대한 주의

`Decision(t) = f(State(t))`는 의사결정이 오직 현재 상태에만 의존하는 것으로 표현되어 있으나, 실제로 의사결정은 과거 이력(history), 미래 예측(forecast), 외생 변수(exogenous factors)에도 의존한다. 이 공식은 문자적 모형이 아니라 설계 원리를 표현하는 휴리스틱(heuristic)으로 이해되어야 한다. 보다 정밀한 정식화는 다음과 같다:

```
Decision(t) = f(State(t), History(0:t-1), Forecast(t+1:T), E(t); θ)
```

그러나 이 확장된 공식에서도 State(t)가 핵심 독립변수라는 점은 변하지 않으며, History와 Forecast는 State(t)의 해석을 풍부하게 하는 보조 변수로 위치한다.

## 4. 레이어 간 간극 및 중복 검토

### 4.1 잠재적 간극 (Potential Gaps)

**간극 1: 거버넌스 레이어의 부재**
SAPD의 5개 레이어는 기술적·설계적 측면을 다루지만, 정치적 의사결정, 이해관계자 갈등 조정, 민주적 정당성 확보 등의 거버넌스 측면이 독립 레이어로 분리되어 있지 않다. 이러한 요소는 L3(의사결정 아키텍처)에 내포되어 있으나, 공공 정책의 맥락에서 이를 독립적으로 다루는 것이 타당할 수 있다.

**대응**: SAPD는 민주적 책임성, 투명성, 형평성을 아키텍처의 설계 제약 조건으로 명시하고 있으며, 이는 모든 레이어에 관통하는 횡단적 원칙(cross-cutting principles)으로 기능한다. 별도 레이어의 추가보다는 각 레이어에 거버넌스 요건을 내재화하는 현재 접근이 아키텍처의 간결성을 유지하면서도 공공 가치를 반영하는 방식이다.

**간극 2: 구현 레이어의 부재**
설계(design)와 운영(operation) 사이의 구현(implementation) 과정—기술 선택, 인력 배치, 조직 변화 관리—이 별도 레이어로 다루어지지 않는다.

**대응**: SAPD는 의사결정 시스템의 '설계론(design theory)'을 자처하며, 구현은 도메인별 맥락에 의존하는 별도의 문제로 간주한다. 이는 건축 설계도가 시공 과정까지 명세하지 않는 것과 유사한 적절한 추상화 수준의 선택이다.

### 4.2 잠재적 중복 (Potential Redundancies)

**중복 가능성: L1과 L5의 기능적 중첩**
L1(문제 구조화)과 L5(평가 피드백)는 모두 시스템 상태에 대한 인식을 다룬다. L1은 초기 상태 인식이고, L5는 실행 후 상태 인식이다. 팔란티어에서 온톨로지가 L1과 L5를 동시에 관통하는 것은 이 중첩의 기술적 반영이다.

**평가**: 이 중첩은 중복이 아니라 순환 구조의 필연적 결과이다. L1은 시스템의 초기 정의를 담당하고, L5는 그 정의의 갱신 가능성을 평가한다. 양자는 기능적으로 구별되지만 대상(시스템의 상태와 경계)을 공유하며, 이 공유가 순환 피드백의 매개체이다.

## 5. 종합 정합성 판정

### 5.1 판정 결과 요약

| 검증 항목 | 판정 |
|----------|------|
| L1 → L2 논리적 흐름 | 정합적 |
| L2 → L3 논리적 흐름 | 정합적 |
| L3 → L4 논리적 흐름 | 정합적 |
| L4 → L5 논리적 흐름 | 정합적 |
| L5 → L1 순환 완결 | 정합적 |
| 핵심 공식 ↔ 아키텍처 정합성 | 정합적 |
| 치명적 간극 존재 여부 | 없음 (보완 가능한 잠재적 간극 2건) |
| 불필요한 중복 존재 여부 | 없음 (순환 구조에 의한 기능적 중첩 1건) |

### 5.2 종합 판정

SAPD Framework의 5-Layer 아키텍처는 내적으로 정합적(internally consistent)이다. 각 레이어는 이전 레이어의 출력을 입력으로 수용하는 논리적 흐름을 형성하며, 핵심 공식 `Decision(t) = f(State(t))`의 모든 요소가 아키텍처 내에서 명확한 대응을 가진다. 식별된 잠재적 간극(거버넌스 레이어, 구현 레이어)은 아키텍처의 추상화 수준 선택에 의한 것으로, 정합성을 훼손하는 논리적 모순이 아니라 향후 확장 가능한 설계 여지이다.
