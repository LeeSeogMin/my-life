# 핵심 개념 정의: SAPD Framework의 조작적 정의 체계

## 개요

본 문서는 논문에서 사용되는 모든 핵심 개념을 학술적으로 엄밀하게 정의한다. 각 개념에 대해 (1) 형식적 정의, (2) 조작적 해석, (3) 기존 이론과의 관계를 명시한다.

---

## 1. 상태 변수 (State Variable)

### 형식적 정의
> **상태 변수 S(t)**는 시점 t에서 정책 시스템이 관찰하는 환경 및 시스템 내부 조건의 측정 가능한 값들의 벡터이다.

### 수식 표현
```
S(t) = {s₁(t), s₂(t), ..., sₙ(t)}
```
여기서 sᵢ(t)는 시점 t에서의 i번째 상태 변수의 값이며, n은 시스템이 추적하는 상태 변수의 총 개수이다.

### 조작적 해석
상태 변수는 정책 시스템이 의사결정을 내리기 위해 관찰하는 현실 세계의 측정값이다. 예를 들어 실업률, 대기질 지수, 감염병 발생 건수, 재정 지출 현황, 민원 접수량 등이 정책 영역에 따른 상태 변수에 해당한다. 핵심적 속성은 다음과 같다:

- **연속성(Continuity)**: 상태 변수는 특정 시점의 스냅샷이 아닌 시간 축 위의 연속적 관찰값이다.
- **측정 가능성(Measurability)**: 상태 변수는 데이터로 포착 가능해야 한다.
- **관련성(Relevance)**: 의사결정에 실질적으로 영향을 미치는 변수만이 상태 변수로 선정된다.

### 기존 이론과의 관계
- **사이버네틱스(Ashby, 1956)**: 제어 시스템에서의 '시스템 상태(system state)' 개념을 계승하되, 공공 정책 맥락으로 구체화한 것이다.
- **적응적 관리(Holling, 1978)**: 모니터링 대상 지표와 개념적으로 유사하나, SAPD의 상태 변수는 실시간 연속 관찰을 전제로 한다는 점에서 차이가 있다.
- **DAPP(Haasnoot et al., 2013)**: DAPP의 '전환 신호(signpost)'가 사전 정의된 이산적 임계값인 반면, SAPD의 상태 변수는 연속적 값을 갖는 벡터이다.

## 2. 의사결정 변수 (Decision Variable)

### 형식적 정의
> **의사결정 변수 D(t)**는 시점 t에서 의사결정자가 제어할 수 있는 정책 수단(policy instrument)의 값이다.

### 수식 표현
```
D(t) = {d₁(t), d₂(t), ..., dₘ(t)}
```
여기서 dⱼ(t)는 시점 t에서의 j번째 의사결정 변수의 값이며, m은 의사결정자가 제어 가능한 변수의 총 개수이다.

### 조작적 해석
의사결정 변수는 정책 시스템 내에서 의사결정자가 실제로 조정할 수 있는 정책 레버(policy lever)이다. 예산 배분 비율, 규제 강도, 서비스 제공 수준, 인력 배치, 금리, 보조금 규모 등이 이에 해당한다. 핵심적 속성은 다음과 같다:

- **제어 가능성(Controllability)**: 의사결정자의 권한 범위 내에서 실제로 조정할 수 있는 변수이다.
- **실행 가능성(Actionability)**: 변수의 변경이 실제 정책 행동으로 전환될 수 있어야 한다.
- **영향력(Impact)**: 의사결정 변수의 변경이 상태 변수에 측정 가능한 영향을 미쳐야 한다.

### 기존 이론과의 관계
- **사이버네틱스(Beer, 1972)**: 생존가능시스템모델에서의 '조절 장치(regulatory mechanism)' 개념과 대응된다.
- **RDM(Lempert et al., 2003)**: RDM의 '정책 레버(policy lever)' 개념과 동일한 대상을 지칭하나, SAPD에서는 상태 함수의 출력으로 정식화된다.

---

## 3. 의사결정 규칙 (Decision Rule)

### 형식적 정의
> **의사결정 규칙 f**는 상태 변수 S(t)를 입력으로 받아 의사결정 변수 D(t)를 출력으로 산출하는 함수이다.

### 수식 표현
```
D(t) = f(S(t))
```
여기서 f는 상태 공간(state space)에서 의사결정 공간(decision space)으로의 매핑 함수이다.

### 조작적 해석
의사결정 규칙은 "주어진 상태에서 어떤 결정을 내릴 것인가"를 규정하는 논리이다. 이 함수 f의 내부는 다양한 형태를 가질 수 있다:

- **규칙 기반(Rule-based)**: If-Then 조건문 형태의 명시적 규칙 (예: "실업률이 X%를 초과하면 고용 지원 예산을 Y% 증가")
- **모델 기반(Model-based)**: 예측 모델의 출력에 기반한 의사결정 (예: 수요 예측 모델에 따른 자원 배분)
- **학습 기반(Learning-based)**: AI/ML 알고리즘에 의한 패턴 인식과 최적화 제안
- **숙의 기반(Deliberative)**: 인간 의사결정자의 판단, 가치 고려, 정치적 숙의를 포함하는 복합적 과정

중요한 점은, 의사결정 규칙 f가 순수 알고리즘이 아니라는 것이다. f는 기술적 계산, 인간 판단, 정치적 고려, 공공 가치를 모두 포함하는 복합 함수이며, SAPD는 이 함수의 구조를 명시적으로 설계하는 것을 목표로 한다.

### 기존 이론과의 관계
- **사이버네틱스(Ashby, 1956)**: 제어 함수(control function) 개념을 직접적으로 계승한다.
- **적응적 관리(Walters, 1986)**: 능동적 적응관리의 '의사결정 모델(decision model)'과 대응되나, SAPD에서는 실시간 상태 입력을 전제로 한다.
- **EBP**: EBP에서의 의사결정은 증거 보고서에 기반한 비형식적 과정인 반면, SAPD에서는 이를 명시적 함수로 정식화한다.

## 4. 시간 지연 (Time Lag)

### 형식적 정의
> **시간 지연 τ**는 정책 시스템의 상태 변화 시점과 해당 변화에 대한 정책적 대응이 실행되는 시점 사이의 시간 간격이다.

### 수식 표현
```
τ = t_response - t_change
```
여기서 t_change는 상태 변화가 발생한 시점, t_response는 정책적 대응이 실행된 시점이다.

### 조작적 해석
시간 지연은 정책 시스템의 반응 속도를 측정하는 핵심 지표이다. EBP 모델에서의 시간 지연은 다음과 같은 하위 지연들의 합으로 구성된다:

```
τ_EBP = τ_detection + τ_collection + τ_analysis + τ_reporting + τ_deliberation + τ_implementation
```

- **τ_detection**: 상태 변화 감지 지연
- **τ_collection**: 데이터 수집 지연
- **τ_analysis**: 분석 및 평가 지연
- **τ_reporting**: 보고서 작성 및 전달 지연
- **τ_deliberation**: 의사결정 숙의 지연
- **τ_implementation**: 정책 실행 지연

SAPD는 이 중 τ_detection, τ_collection, τ_analysis, τ_reporting을 실시간 상태 관찰과 자동화된 의사결정 지원을 통해 구조적으로 최소화하며, τ_deliberation과 τ_implementation은 의사결정 아키텍처(Layer 3)의 설계를 통해 단축한다.

### 기존 이론과의 관계
- **EBP 비판 문헌**: 시간 지연은 EBP에 대한 다양한 비판에서 암묵적으로 언급되나, SAPD에서는 이를 형식적으로 정의하고 분해하여 구조적 해법의 대상으로 명시화한다.
- **사이버네틱스**: 피드백 지연(feedback delay) 개념과 직접적으로 대응된다.

## 5. 적응형 피드백 (Adaptive Feedback)

### 형식적 정의
> **적응형 피드백**은 정책 시스템이 실행 결과를 상태 변수로 재기록하고, 이를 통해 의사결정 규칙 f를 지속적으로 갱신하는 순환적 학습 과정이다.

### 수식 표현
```
State(t+1) = g(State(t), Decision(t))
f(t+1) = h(f(t), State(t+1), Performance(t))
```
여기서 첫 번째 식은 의사결정이 상태를 갱신하는 과정, 두 번째 식은 갱신된 상태와 성과 정보에 기반하여 의사결정 규칙 자체가 진화하는 과정이다.

### 조작적 해석
적응형 피드백은 SAPD 아키텍처의 Layer 5(평가 및 적응형 피드백)에서 구현되며, 두 가지 수준의 학습을 포함한다:

- **1차 학습(Single-loop Learning)**: 현재의 의사결정 규칙 f 하에서 상태 변화에 따라 의사결정 변수 D(t)를 조정하는 것. 규칙은 고정되고 출력만 변한다.
- **2차 학습(Double-loop Learning)**: 의사결정 규칙 f 자체를 수정하는 것. 기존 규칙이 일관되게 부적절한 결과를 산출할 때 규칙의 구조를 변경한다.

EBP에서의 평가가 정책 주기의 '마지막 단계'에 위치하는 반면, SAPD에서의 적응형 피드백은 상태 관찰과 동시에 작동하는 '상시적 과정(continuous operation)'이다.

### 기존 이론과의 관계
- **적응적 관리(Holling, 1978; Walters, 1986)**: 모니터링-평가-조정의 순환 학습 개념을 계승하되, 실시간 연속성을 추가한다.
- **조직 학습 이론(Argyris & Schon, 1978)**: 1차 학습/2차 학습의 구분을 차용한다.
- **사이버네틱스(Ashby, 1956)**: 피드백 제어(feedback control)의 원리를 직접적으로 계승한다.

## 6. 구조적 동형성 (Structural Isomorphism)

### 형식적 정의
> **구조적 동형성**은 두 체계 A와 B 사이에 구성 요소와 관계의 일대일 대응 함수 φ가 존재하여, A에서의 구성 요소 간 관계가 B에서의 대응 구성 요소 간 관계로 보존되는 구조적 속성이다.

### 수식 표현
```
φ: A → B
∀ (aᵢ, aⱼ) ∈ R_A ⟹ (φ(aᵢ), φ(aⱼ)) ∈ R_B
```
여기서 R_A와 R_B는 각각 체계 A와 B 내에서의 구성 요소 간 관계 집합이다.

### 조작적 해석
본 연구에서 구조적 동형성은 SAPD Framework(체계 A)와 팔란티어 플랫폼(체계 B) 사이에 적용된다. 구체적으로:

- SAPD의 5개 레이어(aᵢ)와 팔란티어의 기술 요소(φ(aᵢ))가 일대일 대응
- SAPD 레이어 간의 기능적 관계(R_A)가 팔란티어 기술 요소 간의 기술적 관계(R_B)로 보존됨
- 순환 피드백 루프의 구조가 양 체계에서 동일한 패턴으로 관찰됨

중요한 점은, 구조적 동형성이 '동일성(identity)'이 아니라 '구조의 대응(correspondence of structure)'을 의미한다는 것이다. 두 체계의 맥락, 시간 해상도, 가치 체계는 상이할 수 있으나, 아키텍처의 구조적 패턴이 대응된다.

### 기존 이론과의 관계
- **수학적 구조주의**: 대수학(algebra)에서의 동형성(isomorphism) 개념을 사회과학적 분석에 차용한 것이다.
- **시스템 이론(von Bertalanffy, 1968)**: 일반시스템이론에서 상이한 도메인의 시스템들이 동일한 구조적 원리를 공유할 수 있다는 관점과 일맥상통한다.
- **비교 제도 분석**: 제도적 동형성(institutional isomorphism, DiMaggio & Powell, 1983)과는 구분된다. 제도적 동형성이 조직 간의 형태적 수렴을 다루는 반면, 본 연구의 구조적 동형성은 이론적 아키텍처와 기술적 플랫폼 간의 구조적 대응을 다룬다.

## 7. SAPD 5단계 아키텍처 레이어 (SAPD 5-Layer Architecture)

### 형식적 정의
> **SAPD 5단계 아키텍처**는 상태 기반 적응형 정책의 구현을 위한 구조적 설계 명세로, 5개의 기능적 레이어로 구성된 순환적 시스템이다.

### 각 레이어의 정의

| 레이어 | 명칭 | 기능 정의 | 핵심 질문 |
|--------|------|----------|----------|
| L1 | 문제 구조화 (Problem Structuring) | 시스템의 경계와 정책 문제를 정의하며, 상태 변수의 범위를 설정한다 | "무엇이 문제이며, 시스템의 경계는 어디까지인가?" |
| L2 | 전략 구조화 (Strategy Structuring) | 전략적 구조와 성과 생성 논리를 정의하며, 목표 상태(desired state)를 설정한다 | "어떤 방향으로, 어떤 논리로 변화를 이끌 것인가?" |
| L3 | 의사결정 아키텍처 (Decision Architecture) | 의사결정 노드, 권한, 기준을 정의하며, 의사결정 규칙 f의 구조를 설계한다 | "누가, 어떤 기준으로, 어떤 절차에 따라 결정하는가?" |
| L4 | 데이터 통합 (Data Integration) | 실시간 데이터와 AI를 시스템에 통합하며, 상태 변수의 관찰 인프라를 구축한다 | "어떤 데이터가, 어떻게 의사결정에 연결되는가?" |
| L5 | 평가 및 적응형 피드백 (Evaluation & Adaptive Feedback) | 지속적 평가와 적응 메커니즘을 제공하며, 시스템의 학습 순환을 완성한다 | "시스템이 결과로부터 무엇을 학습하며, 어떻게 진화하는가?" |

### 레이어 간 관계
5개 레이어는 선형적 위계가 아닌 순환적 상호작용 구조를 형성한다. Layer 5의 피드백은 Layer 1의 문제 재정의를 촉발할 수 있으며, 이는 시스템의 완전 순환(full cycle)을 형성한다.

## 8. 핵심 공식 체계 요약

```
핵심 상태 전이 공식:
  State(t) → Decision(t) = f(State(t)) → System Response → State(t+1)

구성 요소:
  S(t) = {s₁(t), s₂(t), ..., sₙ(t)}     [상태 변수 벡터]
  D(t) = {d₁(t), d₂(t), ..., dₘ(t)}     [의사결정 변수 벡터]
  f: S → D                                 [의사결정 규칙]
  g: (S, D) → S                            [상태 전이 함수]
  h: (f, S, Performance) → f               [규칙 진화 함수]

시간 지연:
  τ = t_response - t_change                 [총 시간 지연]
  τ_EBP = Σ(τ_detection + τ_collection + τ_analysis + τ_reporting + τ_deliberation + τ_implementation)

구조적 동형성:
  φ: SAPD_Layers → Palantir_Components     [레이어 대응 함수]
  R_SAPD ≅ R_Palantir                       [관계 구조 보존]
```
